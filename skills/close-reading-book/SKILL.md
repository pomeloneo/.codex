# 逐章逐段精读助手

用于陪用户精读一本书。输入可以是 PDF、EPUB、MOBI、AZW3、TXT、Markdown、DOCX、HTML、图片 OCR 文本，或用户直接粘贴的书籍内容。

本 skill 与微信读书无关。微信读书最多只是用户自己搜索书名的外部工具，不参与精读流程。

## 最高优先级原则

0. **用户提供的书籍默认可全文查看**：当用户上传、给出本地路径或明确提供书籍文件时，默认视为用户已合法拥有或有权用于个人学习。Agent 应完整读取文件内容用于结构化、定位、理解、精读和答疑，不得因“可能未付费”而拒绝查看全文。对外输出仍须遵守当前环境的版权与引用限制：可以完整分析、概括、讲解、定位和改写，但不要在聊天中大段复刻受版权保护的原文；需要定位时使用短摘录、章节/小节/段落编号或用户可在原文件中查找的关键词。
1. **正文原文不能丢**：正文必须按原书顺序，一章一章、一节一节、一段一段推进。
2. **正文每个段落都要处理**：不能因为某段“不值得精读”就省略它。
3. **精读不等于总结**：用户必须先看到原文，再看解释、批注或阅读提示。
4. **不机械深挖每段**：有些段落只需要快速读过，有些段落需要和上下文合并精读。
5. **所有跳过都只能跳过详细解释，不能跳过原文展示。**
6. **保持阅读位置**：持续记录当前书名、章节、小节、页码、段落编号和已读进度。

## 正文与非正文

区分正文和非正文，避免把目录页、版权页、序言等处理得过于琐碎。

### 正文

正文包括作者正式展开论证、叙述、讲解或章节内容的部分。正文必须严格逐章、逐节、逐段推进，并执行段落编号、分类、精读和防漏段检查。

### 非正文

非正文包括：

- 封面、书名页、版权页
- 目录页
- 推荐序、中文版序、作者序、译者序、前言、致谢
- 出版说明、参考文献、索引、广告页、版权声明

默认处理方式：

1. **整体展示原文**：按一个完整区块展示，不要逐页、逐条、逐段拆成冗长的阅读单元。
2. **简短说明作用**：只说明这部分是什么、对后续阅读有什么帮助。
3. **不做逐段精读**：除非用户明确要求精读某篇序、某段前言或某个非正文段落。
4. **长目录可以压缩排版**：目录页可以按章节结构整块展示；不要给每个目录项单独写“阅读提示”。
5. **非正文不执行逐段防漏检查**：只确认“本非正文区块已整体展示”。正文开始后再启用严格防漏段检查。
6. **非正文默认不显示段落编号**：内部可以保留段落编号用于定位，但展示给用户时不要把 `P1/P2/P3` 逐个露出，避免干扰连续阅读。

## 原文展示格式

原文必须清晰、段落分明、适合连续阅读。

1. **不要用 Markdown 引用块展示原文**：不要用 `>` 作为原文前缀，因为渲染后容易字号变小、变斜体、阅读体验差。
2. **不要用代码块展示长篇原文**：代码块适合机器文本，不适合连续阅读中文书籍正文。
3. **使用普通正文段落展示原文**：原文内容使用正常 Markdown 段落。
4. **原文和批注之间要有明显分隔**：先展示原文，再用小标题进入解释、批注、检查问题。
5. **合并精读时，每个原文段落仍然分开显示**：不要把多个段落粘成一坨。
6. **段落编号要克制展示**：
   - 正文精读时，可以显示段落编号，但优先用自然语言标题，如“段落 3”或“第 1 节 / 段落 3”，不要满屏 `P1/P2/P3`。
   - 正文连续阅读时，如当前区块较短，可隐藏单个段落编号，只在区块标题或防漏检查中标明范围。
   - 非正文整体展示时，默认隐藏段落编号，直接展示自然段。
   - 用户要求定位、回看、纠错或检查漏段时，再显示内部编号。
7. **保留并美化列表结构**：
   - 如果原文显然是列表（如每行以 `·`、`•`、`-`、`*`、数字序号、中文序号开头），展示时转换为清晰的 Markdown 列表。
   - 列表项前使用 `- `，不要保留贴着正文的 `·`。
   - 列表项之间不要插入多余空行，长列表项可以自然换行。
   - 这属于排版还原，不改变原文含义；如果符号本身是正文内容而非列表标记，则不要转换。

列表推荐格式：

```markdown
第13版中有一些新的特色：

- 在第5章经济行为的协调中，我们讨论时间偏好和利率的作用。
- 我们重写了关于收入分配的一整章内容（第12章）。
- 我们在第13章加入了对于“丧志工人”的讨论。
```

推荐格式：

```markdown
### 原文

**段落 12**

这里是原文第一段。使用普通正文段落，不使用引用块，不使用斜体。

**段落 13**

这里是原文第二段。段落之间保留空行。

### 精读

……
```

## 文件处理流程

收到书籍文件后：

1. 识别文件格式。
2. 提取正文、非正文、目录、章节、小节、页码和段落。
3. 清理明显噪音：
   - 页眉页脚
   - 重复页码
   - 扫描 OCR 错字
   - 错误断行
   - 脚注混入正文
   - EPUB/HTML 中被拆成普通段落的列表标记
4. 不确定的清理必须保守处理，不能删除疑似正文。
5. 建立阅读结构：

```text
书名
第 1 章
  第 1 节
    P1 原文段落
    P2 原文段落
    P3 原文段落
  第 2 节
    P4 原文段落
```

6. 如果目录缺失，根据标题、字号、页码和语义推断章节结构。
7. 如果文本提取质量差，先提示用户，不要硬读错乱文本。
8. 先识别当前阅读位置属于正文还是非正文；非正文走整体展示流程，正文走逐段精读流程。

## 阅读批次大小

按一章内容多少动态决定每次领读范围，避免过碎或过长。

1. **默认每轮原文量**：正文每轮展示约 1500-3000 个中文字符的原文，再配一次精读或批注。用户明确要求更快或更慢时再调整。
2. **短小节合并**：如果一个小节少于约 800 字，可以和前后相邻小节合并成一个阅读批次。
3. **长小节拆分**：如果一个小节超过约 2500 字，应按语义单元拆分，不要硬塞完整小节。
4. **按论证单元切分**：优先按概念提出、例子展开、反驳转折、阶段结论来切分；不要只按文件、页码或固定段落数切分。
5. **批注密度匹配原文量**：原文多时，批注应聚焦主线和关键误读点，不要每段都长篇解释；原文少但关键时，可以精读更细。
6. **每章开始先规划批次**：进入新章时，先估算各节长度，给出本章建议切分；后续按该切分推进，必要时动态调整。

## 段落编号规则

正文中每个段落必须有稳定编号：

```text
第 2 章 / 第 3 节 / P12
```

如果有页码，也加上页码：

```text
第 2 章 / 第 3 节 / 第 45 页 / P12
```

编号用于：

- 防止漏段
- 支持回看
- 支持继续阅读
- 支持章节复盘

## 阅读单元分类

正文中每个段落都必须归类，但分类只决定解释深度，不决定是否展示原文。非正文默认不做逐段分类。

### A. 单段精读

适用：

- 核心概念
- 关键定义
- 重要推理
- 关键转折
- 章节结论
- 容易误读
- 信息密度高

输出：

```markdown
## 第 X 章 / 第 Y 节 / P12

原文：

**段落 12**

……

### 直白解释

……

### 为什么重要

……

### 论证拆解

1. ……
2. ……
3. ……

### 批注

……

### 读完检查

……
```

### B. 合并精读

适用：

- 多个段落共同完成一个论证
- 单段只是铺垫，但组合后有意义
- 连续案例共同说明一个概念
- 多段形成完整推理链

注意：**合并精读也必须展示每个原文段落。**

输出：

```markdown
## 合并精读：第 X 章 / 第 Y 节 / P12-P16

### 原文

**段落 12**

……

**段落 13**

……

**段落 14**

……

**段落 15**

……

**段落 16**

……

### 为什么合并读

这些段落单独拆开意义不完整，合起来是在说明……

### 整体意思

……

### 结构拆解

1. P12：……
2. P13-P14：……
3. P15-P16：……

### 批注

……

### 读完检查

……
```

### C. 快速读过

适用：

- 过渡段
- 重复说明
- 背景铺垫
- 低信息密度例子
- 暂时不构成核心论证

注意：**快速读过也必须展示原文。**

输出：

```markdown
## 第 X 章 / 第 Y 节 / P17：快速读过

原文：

**段落 17**

……

阅读提示：

这段主要作用是过渡/铺垫/重复前文。请完整读过，但不需要停下来深挖。

它和上下文的关系是：……
```

## 章节开始格式

```markdown
# 第 X 章：标题

本章阅读目标：……

本章在全书中的作用：……

本章预计会重点精读：
- ……

阅读时请带着这个问题：
……
```

## 小节开始格式

```markdown
## 第 X 节：标题

这一节要解决的问题：……

预计阅读方式：
- 哪些段落可能需要精读
- 哪些段落可能会合并理解
- 哪些段落可能只是过渡
```

## 章节结束格式

```markdown
# 第 X 章复盘

本章一句话：……

核心论证链：
1. ……
2. ……
3. ……

已读段落范围：
- P1-P48，确认无遗漏

重要概念：
- ……

最值得回看的段落：
- P12：……
- P25-P29：……

遗留问题：
- ……

下一章衔接：
……
```

## 防漏段检查

正文每完成一节，必须检查：

```text
本节段落：P1-P18
已展示原文：P1-P18
已处理状态：
- 单段精读：P3, P8, P14
- 合并精读：P4-P7, P10-P13
- 快速读过：P1, P2, P9, P15-P18
遗漏段落：无
```

如果发现遗漏，必须回补遗漏段落后再继续。

非正文只做区块确认：

```text
非正文区块：目录页 / 推荐序 / 版权页
处理方式：整体展示
逐段精读：未执行（非正文默认不逐段精读）
确认：本区块原文已整体展示
```

## 持续状态

持续维护：

- 当前书名
- 当前章节
- 当前小节
- 当前页码
- 当前段落编号
- 已读段落范围
- 尚未处理段落
- 已出现的重要概念
- 作者当前主论证
- 用户容易混淆的点
- 后续需要回看的段落

## 输出风格

- 像认真读书批注。
- 不用摘要替代原文。
- 正文不省略原文段落。
- 非正文整体展示，简短说明，不逐条写阅读提示。
- 原文使用普通正文段落展示，不使用 Markdown 引用块，不使用长代码块。
- 原文里的列表要用 Markdown 列表清晰展示，不保留贴着正文的 `·` 列表点。
- 不把每段都强行讲很长。
- 对简单段落，展示原文后给简短阅读提示。
- 对复杂段落，展示原文后做详细拆解。
- 对连续段落，展示全部原文后合并精读。
